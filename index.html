Tutorial

Определяем глобальную переменную.

1 $ export GITHUB_USERNAME=babyrage666
2 $ alias gsed = sed

Подготовка к выполнению Лабораторной работы №6.

1 $ git clone https://github.com/${GITHUB_USERNAME}/lab05 lab07
2 $ cd lab07
3 $ git remote remove origin
4 $ git remote add origin https://github.com/${GITHUB_USERNAME}/lab07

Скачиваем файл catch.hpp, вносим в него изменения и помещаем в каталог test.

1 $ mkdir tests
2 $ wget https://github.com/philsquared/Catch/releases/download/v1.9.3/catch.hpp -O tests/catch.hpp
3 $ cat > tests/main.cpp <<EOF
4 #define CATCH_CONFIG_MAIN
5 #include "catch.hpp"
6 EOF

Редактируем файл CMakeLists.txt.

1 $ sed -i '' '/option(BUILD_EXAMPLES "Build examples" OFF)/a\
2 option(BUILD_TESTS "Build tests" OFF)
3 ' CMakeLists.txt
4 $ cat >> CMakeLists.txt <<EOF
5 
6 if(BUILD_TESTS)
7  enable_testing()
8  file(GLOB \${PROJECT_NAME}_TEST_SOURCES tests/*.cpp)
9  add_executable(check \${\${PROJECT_NAME}_TEST_SOURCES})
10  target_link_libraries(check \${PROJECT_NAME} \${DEPENDS_LIBRARIES})
11  add_test(NAME check COMMAND check "-s" "-r" "compact" "--use-colour" "yes")
12 endif()
13 EOF

Создаем файл test1.cpp и вносим в него изменения.

1 $ cat >> tests/test1.cpp <<EOF
2 #include "catch.hpp"
3 #include <print.hpp>
4 
5 TEST_CASE("output values should match input values", "[file]") {
6  std::string text = "hello";
7  std::ofstream out("file.txt");
8 
9  print(text, out);
10  out.close();
11 
12  std::string result;
13  std::ifstream in("file.txt");
14  in >> result;
15 
16  REQUIRE(result == text);
17 }
18 EOF

Собираем cmake проект.

1 $ cmake -H. -B_build -DCMAKE_INSTALL_PREFIX=_install -DBUILD_TESTS=ON
2 $ cmake --build _build
3 $ cmake --build _build --target test

Обновляем данные в файлах README.md и **.travis.yml**.

1 $ sed -i '' 's/lab05/lab07/g' README.md
2 $ sed -i '' 's/\(DCMAKE_INSTALL_PREFIX=_install\)/\1 -DBUILD_TESTS=ON/' .travis.yml
3 $ sed -i '' '/cmake --build _build --target install/a\
4 - cmake --build _build --target test -- ARGS=--verbose
5 ' .travis.yml

Проверяем файл **.travis.yml** на ошибки.

1 $ travis lint # display warnings for a .travis.yml
2 Warnings for .travis.yml:
3 [x] value for addons section is empty, dropping
4 [x] in addons section: unexpected key apt, dropping

Подтверждаем изменения в проекте и отправляем на удаленный репозиторий сервиса Github.

1 $ git add .
2 $ git commit -m"added tests"
3 $ git push origin master

Подключаем сервис Travis CI

1 $ travis login --auto
2 $ travis enable

Делаем скриншот экрана и помещаем в каталог artifacts.

1 $ mkdir artifacts
2 $ screencapture -T 20 artifacts/screenshot.jpg
